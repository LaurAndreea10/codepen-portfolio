6 issues
LCP
Render-blocking CSS — style.css & Google Fonts
style.css (~18KB) și Google Fonts blochează First Paint. Browserul nu poate randa nimic până nu descarcă și parsează ambele resurse. Pe conexiuni lente → LCP > 3s.
FIX: Extrage CSS-ul critic (~3KB) inline în <head>. Încarcă restul cu rel="preload" + onload. Fonts: adaugă display=swap (deja prezent ✓). Consideră self-hosting.
↑ Potențial economie: ~0.8–1.2s FCP
TBT
JS inline voluminos — main.js + scripts inline în HTML
Multiple <script> inline în body execută înainte ca pagina să fie complet încărcată. Dynamic data engine (~4KB inline) și multiple event listeners blochează main thread.
FIX: Consolidează scripturile inline în main.js (deja defer ✓). Elimină scripturile duplicate din body. Folosește requestIdleCallback pentru boost-urile non-critice.
↑ Potențial economie: ~200–400ms TBT
CLS
Layout shift — fonturi & hero image
Fonturile Google Fonts cauzează FOUT (Flash of Unstyled Text) care produce CLS. Hero image alpis-fusion-demo-poster.svg fără dimensiuni explicite poate cauza reflow la load.
FIX: Adaugă width/height explicit pe toate imaginile. Preload fonturile critice în <head>. Adaugă font-display: swap (deja prezent ✓). Consideră size-adjust fallback font.
↑ CLS target: < 0.1 (estimat actual: ~0.15)
IMG
SVG OG Image fără dimensiuni declarate
alpis-fusion-demo-poster.svg este folosit ca OG image și hero fallback fără width/height pe tag-ul img. SVG-urile fără dimensiuni intrinseci cauzează reflow.
FIX: <img width="1200" height="630" src="...">. Adaugă loading="lazy" pe hero iframe. Consideră conversie la WebP pentru non-SVG images.
↑ Elimină reflow + îmbunătățire CLS
CSS
CSS duplicat — reguli inline + style.css
Regulile pentru .now-tabs, .scan-panel, .badge-wip sunt definite atât în style.css cât și în blocuri <style> inline din HTML. Payload CSS total crescut inutil.
FIX: Mută toate regulile inline în style.css. Elimină blocurile <style> din body. Folosește PurgeCSS sau manual audit pentru clase neutilizate.
↑ Reducere ~15–20% CSS payload
ARIA
Intro overlay — aria-hidden + reduced motion
Intro overlay are aria-hidden="true" corect, dar CSS-ul intro ocupă ~8KB inline în head. Funcționează pe reduced-motion, dar CSS-ul nu este eliminat din DOM la skip imediat.
FIX: CSS-ul intro poate fi mutat în intro.css și încărcat condiționat (skip dacă sessionStorage key prezent). Economie: ~8KB din first-paint critical path.
↑ Potențial: elimină 8KB CSS din critical path pe return visits

Action plan
STEP 1 Inline Critical CSS în <head>

Extrage CSS-ul necesar pentru above-the-fold (nav, hero, hero-card, eyebrow, h1, btn). Pune-l direct în <style> în head. Restul — încarcă async.

/* În <head> — ÎNAINTE */ <link rel="stylesheet" href="style.css"> /* În <head> — DUPĂ */ <style> /* Critical CSS inline — ~3KB */ :root { --bg: #071226; --accent: #4f8cff; ... } .topbar { position: sticky; ... } .nav { ... } .hero { ... } .hero-card { ... } h1 { ... } .btn { ... } </style> /* Non-critical — async */ <link rel="preload" href="style.css" as="style" onload="this.onload=null;this.rel='stylesheet'"> <noscript><link rel="stylesheet" href="style.css"></noscript>
STEP 2 Elimină CSS duplicat inline

Mută toate blocurile <style> din body în style.css. Clasele .now-tabs, .badge-wip, .scan-panel sunt definite de 2 ori.

# Audit CSS duplicat — căutare rapidă în terminal grep -n ".now-tab" index.html # → găsit în <style id="intro-css"> și în style.css # Script bash: extrage toate blocurile <style> din body grep -o '<style>.*?</style>' index.html | wc -c # → estimat ~8KB de CSS duplicat sau inline nenecesar
STEP 3 Intro CSS — condiționat pe return visits

CSS-ul intro (~8KB) este în critical path chiar și pe return visits unde intro-ul nu se mai arată. Mută-l în fișier separat și încarcă-l condiționat.

<script> // Încarcă CSS intro doar la prima vizită if (!sessionStorage.getItem('la_intro_seen')) { const link = document.createElement('link'); link.rel = 'stylesheet'; link.href = 'intro.css'; document.head.appendChild(link); } </script>
